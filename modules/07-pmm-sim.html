pmm-sim：私有 AMM 的解剖台

同一个合约、同一个流动性池、相同的金额——但不同身份的调用者会拿到不同的汇率。pmm-sim 让你在本地测量这个差距，一行 mainnet 交易都不需要发。

场景还原

你用 Jupiter 把 15,000 USDC 换成 WSOL。你的朋友直接调用同一个 HumidiFi 合约做同样的兑换。她拿到的 WSOL 更少。

同一个合约，同一个池子，同一笔金额——为什么结果不同？

答案在于 私有 AMM（Prop AMM）的设计哲学：它们不是中立的。它们识别调用者的身份，并为白名单聚合器提供优惠汇率。pmm-sim 让你量化这个差距。

Solana 的秘密菜单

公开的 AMM（如 Raydium、Orca）用公开的数学公式定价，任何人得到相同的价格。私有 AMM 则不同——它们的定价逻辑是闭源的，并且会根据 CPI 调用者的程序地址给出不同的报价。

HumidiFi
支持 swap-v1 / v2 / v3 三种指令，5 个 WSOL-USDC 市场，对 Jupiter 的报价显著优于直接调用。
SolFi V2
流动性深厚的 WSOL-USDC 和 USDT-USDC 市场，使用预言机定价，对 DFlow 有特别优惠。
GoonFi / BisonFi
新兴私有 AMM，部分对黑名单地址有限制，白名单聚合器享受更低滑点。
Tessera / ObricV2 / ZeroFi
专注于稳定币或特定交易对，ObricV2 使用多路预言机，ZeroFi 有独立的 vault 结构。
四大聚合器的"VIP 通行证"

pmm-sim 支持伪装成 Jupiter（route_v2 指令）、DFlow（swap2 指令）、Titan（swap_route_v2 指令）、OKX Labs（swap_v3_with_cpi_event 指令）。不同聚合器对应不同的路由程序 ID，私有 AMM 通过识别这个 ID 来决定给哪档报价。

两种本地 SVM：pmm-sim vs Delorean

pmm-sim 和 Delorean 都在本地运行 SVM，但它们的出发点截然不同。

Delorean 方式
// 从 Penrose 拉取历史 fixture
let fixture = get_transaction_fixture(
    rpc, &signature
)?;
// 用历史状态构建 MockBank
let bank = build_mock_bank(&fixture);
// 重放那笔交易
execute_transaction(&bank, &fixture.transaction);
pmm-sim 方式

Delorean 是考古学家——还原过去某笔真实交易的完整现场，每个账户状态、每个系统变量都来自历史快照，目的是重放和调试。

pmm-sim 是实验室——从零搭建一个干净的 LiteSVM 环境，注入你需要的账户和程序，然后反复执行 swap，测量汇率和算力消耗。没有历史包袱，随时重置，随时对比。

env.rs — 环境初始化
let mut svm = LiteSVM::new()
    .with_default_programs()
    .with_sysvars()
    .with_sigverify(true)
    .with_compute_budget(budget);

// 注入代币 mint
mints.iter().for_each(|(mint, dec)| {
    svm.set_account(*mint,
        Misc::mk_mint_acc(*dec))
});

// 加载程序（从磁盘或 RPC 实时拉取）
svm.add_program_from_file(
    program_id, path_to_elf
);
中文释义

LiteSVM::new() — 创建一个空白的进程内虚拟机，不需要启动本地验证节点，毫秒级就绪。

with_sysvars() — 注入 Solana 系统变量（rent、clock 等），让程序以为自己在真实网络上运行。

svm.set_account() — 手动插入任何账户状态。可以用磁盘缓存的静态快照，也可以通过 RPC 实时拉取 mainnet 最新状态（JIT 模式）。

add_program_from_file() — 加载私有 AMM 的 ELF 字节码。程序本身是公开可下载的，即使逻辑闭源。

CPI 伪装：穿上 Jupiter 的外衣

私有 AMM 通过检查 CPI 调用链里的 program_id 来判断调用者身份。pmm-sim 的核心技巧：把 Magnus Router 的 program_id 替换成目标聚合器的 program_id，让私有 AMM 以为是 Jupiter 或 DFlow 在调用它。

builder.rs — 指令构建器
pub fn instruction(
    &self,
    spoof: Option<Aggregator>,
    data: SwapArgs,
) -> Instruction {
    match spoof {
        None =>
            build_ix!(SwapBuilder),
        Some(Aggregator::Jupiter) =>
            build_ix!(RouteV2Builder),
        Some(Aggregator::DFlow) =>
            build_ix!(Swap2Builder),
        Some(Aggregator::Titan) =>
            build_ix!(SwapRouteV2Builder),
        Some(Aggregator::OkxLabs) =>
            build_ix!(SwapV3WithCpiEventBuilder),
    }
}
中文释义

spoof: None — 用 Magnus Router 本身的身份调用，私有 AMM 按"普通来源"定价。

Aggregator::Jupiter → RouteV2Builder — 使用 Jupiter 的 route_v2 指令鉴别符（discriminator），并把 program_id 替换成 Jupiter 的地址。私有 AMM 看到"Jupiter 在调用我"，切换到优惠汇率档。

每个聚合器对应不同指令 — 这是关键。私有 AMM 不只是检查 program_id，它们还检查指令的鉴别符（前 8 字节）。每个聚合器的接口略有不同，必须完整仿冒才能触发优惠档位。

结果：同一笔 swap，用 --spoof=jupiter 和 --spoof=dflow 各跑一次，就能量化两个聚合器在同一个私有 AMM 里的汇率差距。

基准测试：批量测量汇率曲线

单次 swap 告诉你一个价格点。benchmark 模式让你扫描整条汇率曲线——从 1 WSOL 到 4000 WSOL，每步 1 WSOL，测量每个规模下的汇率和 CU 消耗，结果保存为 parquet 文件，可用 plot.py 可视化。

bash benchmark 示例 
# 基准测试：humidifi、tessera、solfi-v2、goonfi
# 范围 1-4000 WSOL，步长 1 WSOL，伪装为 Jupiter
pmm-sim benchmark \
  --pmms=humidifi,tessera,solfi-v2,goonfi \
  --range=1.0,4000.0,1.0 \
  --src-token=wsol --dst-token=usdc \
  --spoof=jupiter \
  --jit-accounts=false  # 用磁盘缓存的静态账户

# 用 DuckDB 快速查看结果
duckdb -csv \
  -c "SELECT * FROM 'datasets/401101387_jupiter_humidifi_*.parquet'" \
  | column -t -s ,

PropAMM 出海：为什么 EVM 之前做不到

pmm-sim 所量化的这套机制——实时报价、身份识别、私有汇率——是 Solana 特有的产物。当同样的逻辑遇到以太坊的基础设施，会撞上一堵墙。

Solana — 为什么能

无公开 mempool：交易直接发给领导者，做市商更新报价前没有人能抢跑。

微交易成本极低：每笔报价更新几乎免费，每秒可更新数百次而不亏损。

Jito 优先通道：高价值交易可以买到确定性的区块排序，MEV 可预测。

EVM — 为什么难

公开 mempool = 毒流量：套利者看到你的报价更新交易，可以在同一区块里抢先执行。每次更新都是暴露。

更新成本高：以太坊 gas 费让高频报价更新经济上不可行。

状态更新延迟：12 秒出块周期让"实时"报价实为历史数据。

BaiBai：第一个把 PropAMM 带到 Base 的平台

2025 年，BaiBai（基于 Spire Labs 的 Pylon 构建）提出了一套破局方案，让 PropAMM 的经济模型在 EVM 上首次成立：

①
做市商专属区块空间
报价更新交易保证排在每个区块最顶部——做市商更新报价时，不会被套利者在同一区块内抢跑。
②
对毒流量设置速度限制
检测到已知套利地址后，自动添加小延迟（speed bump），让做市商有窗口先撤销旧报价。
③
同步可组合性 + 智能路由
与 Base 原生同步组合，无需跨链桥。未被 BaiBai 填满的大额 swap 自动回退到公开流动性（外部 AMM），避免流动性碎片化。
为什么这对你重要

pmm-sim 目前只测量 Solana 上的私有 AMM。但它所量化的身份敏感定价范式，正在成为多链 DeFi 的基础设施。理解 CPI 伪装、优惠档汇率、基准数据格式——这些知识在 BaiBai、在未来的 EVM PropAMM 上都会再次出现。
